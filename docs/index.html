Module Documentation

How It Works

  1. 1 Parses composer.lock, composer.json and auth.json to identify installed packages, private repos and credentials
  2. 2 Matches packages against vendor URL mappings, Packagist registry, and private Composer repositories
  3. 3 Scrapes vendor product pages using vendor-specific regex patterns to extract latest version
  4. 4 Queries private Composer repos (Satis) with auth credentials for vendors like Amasty, MageWorx and XTENTO
  5. 5 Falls back to public Packagist API when website scraping and private repo checks fail
  6. 6 Compares installed vs latest versions and generates the report with source tracking

Version Sources

Website Vendor Website Scraping

Directly scrapes the vendor's product page using HTTP requests and vendor-specific regex patterns. Most accurate but subject to Cloudflare protection and page structure changes.

Packagist Packagist API

Queries the public Packagist registry API. Used as a fallback for website scraping failures and as the primary source for packages not distributed via vendor websites.

Private Repo Private Composer Repository

Queries the vendor's private Composer repository (Satis) using credentials from auth.json. Used for vendors like Amasty, MageWorx, and XTENTO that distribute via private repos. Requires valid project-level credentials.

Failed Cloudflare / Auth Expired

Vendor website blocked by Cloudflare and private repo credentials are expired or missing. Check auth.json credentials and vendor subscription status.

Known Limitations

  • Amasty — Website blocked by Cloudflare. Private Composer repo (composer.amasty.com) returns 403 because global keys were deprecated on 1 Jan 2026. Requires new project-level Composer keys created in the Amasty customer portal. See PHP 8.4 Compatibility Report for full migration steps.
  • Aheadworks — Website blocked by Cloudflare Managed Challenge v3. Private repo (dist.aheadworks.com) credentials present in auth.json but repo format not yet verified.
  • MageWorx — Website does not display version numbers. Private repo (packages.mageworx.com) credentials present but returning errors — auth may be expired or repo format may differ.
  • XTENTO — Private repo (dowlis.repo.xtento.com) is working. Successfully resolving versions for pdfcustomizer and xtcore. Website scraping also works for orderexport.
  • Stripe Payments — The Magento module (stripe/stripe-payments) is distributed via Stripe's private Composer repository, not public Packagist. The Stripe PHP SDK (stripe/stripe-php) is available via Packagist.
  • Website structure changes — Vendor-specific regex patterns may need updating if vendors redesign their product pages.
