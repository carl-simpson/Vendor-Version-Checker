Module Documentation

How It Works

  1. 1 Parses composer.lock to identify installed third-party packages and their versions
  2. 2 Matches packages against known vendor URL mappings and Packagist registry
  3. 3 Scrapes vendor product pages using vendor-specific regex patterns to extract latest version
  4. 4 Falls back to Packagist API when website scraping fails or returns no version
  5. 5 Compares installed vs latest versions and generates the report

Version Sources

Website Vendor Website Scraping

Directly scrapes the vendor's product page using HTTP requests and vendor-specific regex patterns. Most accurate but subject to Cloudflare protection and page structure changes.

Packagist Packagist API

Queries the public Packagist registry API. Used as a fallback for website scraping failures and as the primary source for packages not distributed via vendor websites.

Failed Cloudflare / Private

Vendors behind Cloudflare Managed Challenge (Amasty, Aheadworks) or using private Composer repositories without public Packagist listings cannot be checked automatically.

Architecture
  • ComposerPlugin.php — Plugin entry
  • CommandProvider.php — CLI registration
  • VendorCheckCommand.php — Command logic
  • VersionChecker.php — Scraping engine
  • ComposerIntegration.php — Lock file parsing

Known Limitations

  • Amasty & Aheadworks — Protected by Cloudflare Managed Challenge v3. Requires JavaScript execution which HTTP clients cannot perform. Not available on public Packagist.
  • MageWorx — Product pages do not display version numbers. Not available on public Packagist. Versions must be checked manually via their customer portal.
  • Stripe Payments — The Magento module (stripe/stripe-payments) is distributed via Stripe's private Composer repository, not public Packagist.
  • Website structure changes — Vendor-specific regex patterns may need updating if vendors redesign their product pages.
